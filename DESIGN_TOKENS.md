## Обзор

Дизайн-система построена на основе двух уровней токенов:

- **Примитивные токены (Primitives)** — базовые значения (цвета, числа, шрифты), которые не зависят от контекста использования
- **Семантические токены** — значения, которые описывают назначение и контекст использования (например, `bg-primary`, `fg-secondary`)

Такая архитектура позволяет:
- Легко поддерживать консистентность дизайна
- Быстро адаптировать дизайн под разные темы (Light/Dark)
- Масштабировать систему при добавлении новых компонентов

## Структура токенов

### Иерархия

```
Primitives (Базовые значения)
├── Color (Цвета)
│   ├── Brand (Брендовые цвета)
│   ├── Core (Основные цвета)
│   ├── Core Inverted (Инвертированные основные цвета)
│   ├── Gray (Серые цвета)
│   ├── System (Системные цвета)
│   └── Feedback (Цвета обратной связи)
├── Numbers (Числовые значения)
└── Typography (Типографика)
    └── Family (Семейство шрифтов)
    └── Font Styles (Стили шрифтов)

Semantic Tokens (Семантические токены)
├── Color (Цвета по назначению)
│   ├── Light (Светлая тема)
│   └── Dark (Темная тема)
├── Layout (Макет и типографика)
│   └── Breakpoints (Брейкпоинты: 320, 360, 375, 430)
└── Radius (Радиусы скругления)
```

- Брейкпоинтов

**Правила применения:**

- Не создавайте произвольные значения

#### Dark Theme (Темная тема)

Структура аналогична Light Theme, но значения адаптированы для темного режима.

**Основные отличия:**
- `dark-bg-primary` = `#22263b` (темный фон)
- `dark-fg-primary` = `#ffffff` (светлый текст)
- Инвертированы значения для `system-black` и `system-white`
- Адаптированы альфа-каналы для лучшей читаемости

## Правила применения

### 1. Использование примитивных vs семантических токенов

**Используйте примитивные токены:**
- При создании новых компонентов
- Когда нужен прямой доступ к базовым значениям
- Для кастомизации существующих компонентов

**Используйте семантические токены:**
- В большинстве случаев при разработке интерфейсов
- Для обеспечения правильной работы темной темы
- Для поддержания консистентности дизайна

### 2. Выбор цвета

**Для фонов:**
- `bg-primary` — основной фон страницы
- `bg-secondary` — фон карточек и контейнеров
- `bg-tertiary` — фон секций
- `bg-accent` — акцентные фоны

**Для текста:**
- `fg-primary` — основной текст
- `fg-secondary` — вторичный текст (подзаголовки)
- `fg-tertiary` — третичный текст (метаданные)
- `fg-muted` — приглушенный текст (подсказки)

**Для границ:**
- `border-primary` — основные границы
- `border-secondary` — тонкие границы
- `border-disabled` — границы неактивных элементов

### 3. Работа с темами

**Светлая тема (Light):**
- Используйте токены с префиксом `light-` или без префикса (по умолчанию)
- Фон страницы: `bg-primary` = белый
- Основной текст: `fg-primary` = темный

**Темная тема (Dark):**
- Используйте токены с префиксом `dark-`
- Фон страницы: `dark-bg-primary` = темный
- Основной текст: `dark-fg-primary` = светлый

**Переключение тем:**
```tsx
// Используйте класс 'dark' на корневом элементе
<div className="bg-light-bg-primary dark:bg-dark-bg-primary">
  <p className="text-light-fg-primary dark:text-dark-fg-primary">
    Текст адаптируется под тему
  </p>
</div>
```

### 4. Типографика

**Выбор стиля:**
- Заголовки: `Title-XL`, `Title-L`, `Title-M`, `Title-S`
- Основной текст: `Body-L`, `Body-M`
- Метки: `Label-L`, `Label-M`, `Label-S`
- Числа: `Number-L`, `Number-S`

**Адаптивность:**
- Размеры шрифтов автоматически адаптируются под брейкпоинты
- Используйте responsive классы Tailwind для контроля

### 5. Отступы и промежутки

**Используйте значения из Numbers:**
- `p-4`, `p-8`, `p-16` — внутренние отступы
- `m-4`, `m-8`, `m-16` — внешние отступы
- `gap-4`, `gap-8`, `gap-16` — промежутки в flex/grid

**Не создавайте произвольные значения!**

### 6. Радиусы скругления

**Выбор радиуса:**
- `rounded-xs` — малые элементы (иконки, бейджи)
- `rounded-s` — небольшие карточки
- `rounded-m` — стандартные карточки
- `rounded-l` — крупные карточки
- `rounded-full` — кнопки, аватары, полностью круглые элементы
